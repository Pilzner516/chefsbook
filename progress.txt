### Root Cause Analysis
The Expo Router doesn't allow both `[id].tsx` and `[id]/index.tsx` in the same parent directory because they resolve to identical route patterns. This caused the app to crash immediately on launch when the route table was built, affecting ALL screens including sign-in. The TypeScript error was a separate issue that would have caused problems with Input components.
