## What to build

Replace the current print stylesheet/component entirely with a properly designed
print layout. This is a `@media print` CSS redesign — do not use PDF generation.

### Layout requirements

**Single column only**
- One department section per column width — full page width
- No multi-column CSS — every item must be fully readable
- This allows the user to physically tear the printed sheet by department
  and hand a section to another person

**Page structure (top to bottom):**

1. **Header**
   - ChefsBook logo (use the existing logo asset) — left aligned, small (24px height)
   - List name (e.g. "Whole Foods") — right aligned, bold, large
   - Date printed — right aligned, small, below list name
   - Thin horizontal rule below header

2. **Department sections**
   - Department name: ALL CAPS, brand red (#C0392B), bold, 13pt, generous top margin
   - Item count in parentheses after department name: "(8 items)"
   - Thin rule below department name
   - Each ingredient row (see below)
   - Extra bottom margin after last item in section to create visual separation
     and a natural tear line

3. **Each ingredient row:**
   ```
   [ ]  2 tablespoons    Butter                    The Pear Pie
   ```
   - Checkbox: a printed square (□) — 14×14pt, left margin, vertically centered
   - Quantity + unit: left column, fixed width, right-aligned number, unit in full
     (tablespoons not T, teaspoons not t, cups not c)
   - Ingredient name: main column, full name — NEVER truncated
   - Recipe source: right-aligned, small, muted — the recipe this ingredient came from
   - Row height: generous — minimum 22pt so the checkbox is easy to tick by hand
   - Alternating very light background on rows (white / #FAFAF8) for readability

4. **Footer**
   - "Generated by ChefsBook — chefsbk.app" — centered, small, muted
   - Page number: "Page X of Y" — right aligned

### Typography for print
- Base font size: 11pt minimum (current is too small)
- Department headers: 13pt bold
- Ingredient names: 11pt
- Quantity/unit: 11pt
- Recipe source: 9pt
- Use a system serif or the brand font — check what is available

### Page breaks
- Add `page-break-before: always` or `break-before: page` before each department
  section IF the department has more than 8 items AND is not the first department
- This keeps each large department on its own page for easier tearing
- Small departments (≤3 items) should NOT force a page break — they can flow together

### Unit expansion
Expand all unit abbreviations to full words in the print view:
- t, tsp → teaspoon / teaspoons
- T, tbsp, Tbsp → tablespoon / tablespoons
- c, C → cup / cups
- oz → ounce / ounces
- lb → pound / pounds
- pkg → package
- pt → pint
- qt → quart

Use singular for quantity = 1, plural otherwise.

### Branding
- Header logo: use the existing ChefsBook logo image asset
- Brand red (#C0392B) for department headers
- No other colours — clean black and white for ink efficiency

### Hide screen-only elements when printing
The following must be hidden in print:
- Navigation sidebar
- Top navigation bar
- Filter/sort buttons (Dept, Recipe, A-Z buttons)
- Font size controls (A- A+)
- The print, refresh, and delete icon buttons
- Any modals or overlays
- Checked-off items should still print (user may want a record)
